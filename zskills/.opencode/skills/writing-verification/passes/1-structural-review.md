# Pass 1: Structural Review

Evaluate document architecture, section roles, logical sequencing, and narrative arc.

## Checklist

- [ ] **Core message identifiable**: Can the reader state the main point in one sentence?
- [ ] **Opening hooks**: First paragraph creates reason to continue reading
- [ ] **Section roles clear**: Each section has a distinct job (introduce, argue, evidence, conclude)
- [ ] **Logical sequence**: Sections appear in an order that builds understanding
- [ ] **Transitions work**: Movement between sections is smooth, not jarring
- [ ] **No orphan sections**: Every section connects to the core argument
- [ ] **Conclusion delivers**: Ending satisfies the promise made by the opening
- [ ] **Appropriate length**: No section is bloated or starved relative to its importance

## Section Role Map

For each major section, identify:

```
Section: [title/number]
Role: [introduce | context | argue | evidence | counter | synthesize | conclude]
Connects to core message: [yes/no — if no, flag]
Transition in: [smooth / abrupt / missing]
Transition out: [smooth / abrupt / missing]
```

## Structural Patterns to Flag

| Issue | Description |
|-------|-------------|
| Buried lede | Core message appears late or is implicit |
| Front-loaded | All substance in first half, second half is filler |
| Missing warrant | Claim made without section providing evidence |
| Circular | Conclusion restates introduction without advancing |
| Tangent | Section that doesn't serve the core argument |
| Abrupt ending | Document stops without resolution |

## Output

```
PASS 1 RESULT: PASS / FAIL
Core message: "[one sentence]"
Sections: [count] — all roles clear: yes/no
Blockers: [list, or "none"]
Warnings: [list, or "none"]
```

A structural incoherence (missing core message, orphan section, or circular argument) is an automatic blocker.
