# Pass 5: Pre-Publish Gate

Final binary pass/fail assessment. This is the last check before publication.

## Gate Checklist

Each item is PASS or FAIL. No "mostly fine."

### Content
- [ ] Core message crystal clear
- [ ] All facts checked (or flagged for author verification)
- [ ] Examples relevant and sufficient
- [ ] No unanswered questions left for the reader

### Structure
- [ ] Opening hooks
- [ ] Flow is logical throughout
- [ ] Conclusion delivers on opening's promise

### Clarity
- [ ] No unexplained jargon
- [ ] No ambiguous pronouns or garden-path sentences
- [ ] Appropriate for target audience

### Style
- [ ] Tone consistent
- [ ] Sentence variety adequate
- [ ] No AI patterns remaining

### Polish
- [ ] Spelling clean
- [ ] Grammar clean
- [ ] Formatting consistent
- [ ] Links work (if applicable)

### Intent
- [ ] Piece achieves its stated purpose
- [ ] Author would be proud to attach their name

## Scoring

Count PASS items out of total. Compute percentage.

- **>=90% PASS**: READY TO PUBLISH
- **75-89% PASS**: PASS WITH NOTES (list the fails for author decision)
- **<75% PASS**: NEEDS REVISION (return to relevant earlier pass)

## Final Verdict

```
PASS 5 RESULT: PASS / FAIL
Score: [X]/[total] ([percentage]%)
Verdict: READY TO PUBLISH / PASS WITH NOTES / NEEDS REVISION
Remaining issues:
1. [issue]
2. [issue]
```

## Rules

1. Run this pass even if earlier passes all passed — fresh-eyes perspective catches things
2. Do not rubber-stamp. If in doubt, FAIL the item
3. A single FAIL does not block publication if the author consciously accepts it
4. But flag it explicitly — author must acknowledge, not overlook
