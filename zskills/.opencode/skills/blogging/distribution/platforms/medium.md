# Medium

Rewrite or adapt a blog post for Medium publication.

## Platform Characteristics

Medium is long-form with a reading-time estimate. Unlike LinkedIn (fold-constrained) or X (char-constrained), Medium rewards depth. But it penalizes fluff — the reading-time indicator sets expectations.

| Parameter | Value |
|-----------|-------|
| Ideal length | 1,000-2,500 words (5-12 min read) |
| Title | SEO-optimized; Medium's internal search uses it |
| Subtitle | Available; use for secondary keyword or hook |
| Tags | Up to 5; choose established tags with large followings |
| Images | Featured image strongly recommended (appears in feed) |
| Code blocks | Supported (use Gists for long code) |
| Formatting | Headers, bold, italic, blockquotes, bullet lists, code |

## Structure

```markdown
# [Title — specific, SEO-aware, 40-60 chars]
## [Subtitle — secondary hook or keyword]

{Featured image}

{Hook — 1-2 sentences, problem or conclusion first}

{Key Takeaways or TL;DR (optional but recommended for long posts)}

## [Section 1: Information heading]
{Answer-first paragraph, then supporting detail}

## [Section 2: Information heading]
...

## [Final section: practical takeaway or call to action]

---

*[Author bio line — one sentence with credentials]*
```

## Medium-Specific Rules

### Tags
- Choose 5 tags from established Medium tags (large follower counts)
- Mix broad (`Programming`, `Data Science`) with niche (`Signal Processing`, `Quantitative Finance`)
- Check existing popular tags — Medium's tag pages show follower counts

### Publications
- Submitting to a publication (Better Programming, Towards Data Science, etc.) dramatically increases reach
- Match the publication's voice and formatting requirements
- Include the publication's preferred heading style

### SEO on Medium
- Medium has high domain authority; titles rank well in Google
- H1 (title) and H2s should contain keywords naturally
- First 150 characters are used as meta description (make them count)
- Medium's canonical URL system means you can cross-post from your own blog without duplicate content penalties

### Reading Time Optimization
- 7-minute posts get the most engagement on Medium (research by Medium's data team)
- If longer, add clear section headings so readers can scan
- If shorter, consider whether it's better as a LinkedIn post instead

### Voice for Medium
- More polished than X or LinkedIn, less formal than academic
- Can be literary — metaphors, narrative structure, personal anecdotes
- First person works well; "we" only if genuinely collaborative
- Humor welcome but must serve the argument

## Adaptation from Blog Post

When adapting an existing blog post for Medium:

1. **Title**: May need to be more provocative for Medium's browse-driven discovery
2. **Opening**: Can be slightly more narrative/literary than the blog version
3. **Code**: Long code blocks → link to Gist; keep inline code short
4. **Images**: Add featured image; ensure all images render in Medium's editor
5. **Internal links**: Replace with Medium-appropriate links (to your other Medium posts or relevant external)
6. **CTA**: Medium readers don't want "subscribe to my newsletter" — end with insight, not sales

## Guardrails

1. No clickbait titles that don't deliver
2. No "follow me for more" CTAs (against Medium culture)
3. No paywall gaming (stuffing keywords to appear in partner program searches)
4. Respect publication guidelines if submitting to one
5. If cross-posting from your blog, set canonical URL to original
6. 5 tags maximum; all must be existing Medium tags
