# D2 Diagrams

Generate professional architecture and system diagrams using [D2](https://d2lang.com/). D2 produces higher-quality layout than Mermaid for complex architecture diagrams, with first-class C4 model support.

## Prerequisites

```bash
which d2 || brew install d2
```

## When to Use D2 (vs Mermaid)

- Standalone architecture diagrams (not inline in markdown)
- Complex graphs (>10 nodes) where layout quality matters
- C4 model diagrams at any level
- When you need SVG output with tooltips/links
- Fan-in/fan-out patterns
- Sequence diagrams with many actors

## C4 Model Framework

### Level Selection

| Situation | Level | Nodes |
|-----------|-------|-------|
| "How does the platform work?" | L1 Context | 3-8 |
| "How do our services connect?" | L2 Container | 5-15 |
| "How does this service handle a request?" | L3 Component | 5-12 |
| "What are the classes?" | L4 Code | Usually skip |

**Default to Level 2 (Container)** unless specified otherwise.

### Edge Colors

| Protocol | Color | Hex |
|----------|-------|-----|
| HTTP / REST | Green | `#16a34a` |
| Pub/Sub / Streaming | Blue | `#2563eb` |
| gRPC | Purple | `#7c3aed` |
| SQL / Database | Amber | `#ca8a04` |
| WebSocket | Cyan | `#0891b2` |
| Background / Scheduled | Gray dashed | `#9ca3af` |
| Feedback / Reverse | Orange dashed | `#ea580c` |

### Node Naming (C4 Convention)

Every node must have a `[bracketed]` annotation:
- `"api-gateway\n[Container: Go]"`
- `"orders-db\n[Container: PostgreSQL]"`
- `"Payment Provider\n[External System]"`

## Workflow

### 1. Plan

- **C4 Level**: Default L2 Container
- **Direction**: `down` for hierarchies, `right` for flows
- **Scope**: One story per diagram. If >10 nodes, split.

### 2. Write D2 Source

Place in `docs/diagrams/<name>.d2`. Follow C4 naming conventions.

### 3. Render

```bash
# SVG (preferred for docs)
d2 --theme 0 --layout elk --pad 60 input.d2 output.svg

# PNG (for chat/presentations)
d2 --theme 0 --layout elk --pad 60 input.d2 output.png

# High-DPI PNG
d2 --theme 0 --layout elk --pad 60 --scale 2 input.d2 output.png
```

- `--theme 0` — clean light theme (use `--theme 200` for C4 look)
- `--layout elk` — best for complex graphs
- `--pad 60` — breathing room

### 4. Evaluate (Judge Loop)

Read the rendered PNG and score each criterion 1-10. **Iterate until ALL >= 8:**

| Criteria | Check |
|----------|-------|
| C4 compliance | Classes from style guide? Tech annotations? Protocol labels? Boundary present? |
| Flow | Arrows one direction? No crossings? |
| Readability | Text legible? Labels not truncated? |
| Grouping | Related nodes close? Visual hierarchy? |
| Simplicity | Every node necessary? Could split? |

### 5. Iterate

If any criterion < 8: reorder nodes, change direction, split diagram, adjust labels. Re-render and re-evaluate.

### 6. Deliver

Commit both `.d2` source and rendered `.svg`.

## Key Principles

1. Follow the framework — use C4 conventions and style guide classes, don't invent custom ones
2. One story per diagram
3. Arrows flow in one direction (dashed for feedback loops)
4. Node declaration order matters (D2/ELK places in declaration order)
5. Protocol in brackets on every edge: `[Pub/Sub]`, `[SQL]`, `[gRPC]`, `[HTTPS]`
6. Architecture, not behavior — diagrams show what connects, not failure modes

## Advanced Features

### Icons

```d2
gateway: "API Gateway" {
  icon: https://icons.terrastruct.com/aws%2FNetworking%20&%20Content%20Delivery%2FAmazon-API-Gateway.svg
}
```

### Layers (Independent Views)

```d2
layers: {
  containers: {
    # Completely separate board (e.g., L2 view)
  }
}
```

### Steps (Sequential Animation)

```d2
steps: {
  1: { client -> api: "1. request" }
  2: { api -> db: "2. query" }
}
```

Export animated: `d2 --animate-interval 1200 flow.d2 flow.svg`

### Sequence Diagrams

```d2
flow: {
  shape: sequence_diagram
  client: "Client"
  api: "API"
  client -> api: "POST /orders\n[HTTPS]"
  api -> client: "201 Created"
}
```

### Grid Layouts

```d2
matrix: {
  grid-columns: 4
  item1: "Adopt" { style.fill: "#d1fae5" }
  item2: "Trial" { style.fill: "#fef9c3" }
}
```

### Legend

```d2
d2-legend: {
  shape: rectangle
  "Go Service" { class: go }
  "Database" { class: db }
}
```

## Layout Engine Comparison

| Engine | Flag | Best for |
|--------|------|----------|
| ELK | `--layout elk` | Complex graphs (default recommendation) |
| Dagre | `--layout dagre` | Simple hierarchies, faster |
| TALA | `--layout tala` | Premium (requires license) |
