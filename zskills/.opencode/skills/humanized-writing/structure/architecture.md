# Document Architecture

Adapted from John McPhee's structural diagramming method.

## Core Principles

1. **Structure arises from material** — Don't impose; let content suggest its organization.
2. **Blueprint before building** — Always plan structure before drafting.
3. **Invisible architecture** — Good structure is invisible. Readers experience flow, not scaffolding.
4. **Three options minimum** — Sketch at least 3 structural options before choosing.
5. **Gold-coin distribution** — Place reader rewards (insights, anecdotes, surprises) throughout, especially in the middle.

## The 8 Structure Types

| Type | Best for | Character |
|------|----------|-----------|
| **List** | Independent points, how-to guides | Simple, clear. Readers can skip around. |
| **Chronological** | Narratives, processes, history | Natural momentum. Can be predictable. |
| **Circular** | Hook with drama, then provide context | Open mid-action → flashback → return. Immediate engagement. |
| **Dual Profile** | One subject, multiple perspectives | Central subject with radiating viewpoints. Multi-dimensional. |
| **Triple Profile** | Complex subject in multiple contexts | Three connected perspectives sharing a protagonist. |
| **Pyramid (Inverted)** | News, exec summaries, busy readers | Most important first, descending order. Frontloaded. |
| **Parallel Narratives** | Comparing/contrasting, multi-threaded | Multiple storylines interwoven. Rich but risky if unclear. |
| **Custom** | Unique material | McPhee-style idiosyncratic diagrams. Whatever serves the piece. |

## Selection Criteria

| Reader type | Best structure |
|-------------|---------------|
| Busy / will skim | Pyramid |
| Engaged / will read all | Circular or Parallel |
| Unfamiliar with topic | Chronological |
| Expert | Dual/Triple Profile |

| Material type | Best structure |
|---------------|---------------|
| Personal narrative | Chronological or Circular |
| How-to guide | List |
| Argument/persuasion | Pyramid or Problem-solution |
| Complex multi-angle story | Parallel |
| Profile/character piece | Dual/Triple Profile |

## Gold-Coin Placement

Gold coins (Roy Peter Clark's metaphor): rewards that keep readers moving forward. Surprising facts, vivid anecdotes, humor, insights, beautiful metaphors.

**Placement rules:**
- **Opening:** One gold coin to hook.
- **Middle:** Most important. Combat the saggy middle. Distribute evenly.
- **End:** Strong gold coin for a satisfying close.
- **Don't front-load** all strong material in the opening.

## Workflow

1. **Gather material** — Understand everything before structuring.
2. **Identify organizing principle** — Time? Space? Importance? Causality? Comparison?
3. **Sketch 3 options** — For each: type, diagram, how material maps, pros/cons.
4. **Test each** — "Does this feel inevitable or forced?"
5. **Select and annotate** — Map gold coins, transitions, pacing.

## Diagrams

You don't need beauty. A sketch is for you. McPhee's diagrams "look like the late-stage wall sketches of a hermit stuck in a cave" — but they work.

```
Chronological:     Day 1 → Day 2 → Day 3 → Day 4
Circular:          Day 5 (action) → Day 1 (flashback) → Day 2,3,4 → Day 5 (resolution)
Parallel:          Thread A: ──•──•──•──
                   Thread B: •──•──•──•
With tangents:     ────○────○────○──  (main line with loops for digressions)
```
